> WHAT IS WHOIS

WHOIS is the registration database behind every domain. When a domain is registered, the registrar submits the registrant name, contact email, nameservers, and expiry date to a central registry.

The whois binary queries these registries directly — different servers for .com (Verisign), .org (AFILIAS), .uk (Nominet), and country-code TLDs.

Because registrants must supply contact data at registration time, WHOIS reveals infrastructure choices made before deployment — registrar tier, NS provider, DNSSEC status, lock configuration.

> WHY THIS SCRIPT

The raw whois output is unstructured text: inconsistent field names, variable date formats, GDPR redactions. No classification, no scoring.

What this script adds:

  • Parses 20+ field variations across registrars
  • Scores risk: domain age, expiry window, lock count, NS type, registrar tier
  • Detects: personal email, single point of failure, server cluster ID in NS hostname
  • SQLite cache — repeat queries instant (24h TTL)
  • Structured JSON for piping

> RISK SCORING — HOW IT WORKS

The risk engine walks every parsed field and assigns weighted penalty points. Score is capped at 100. A domain with multiple compounding signals — newly registered, no locks, personal email, budget registrar — accumulates fast. A well-configured domain (10+ years old, enterprise registrar, 4 locks, DNSSEC signed) scores near zero.

Signal weights:

  • Domain < 7 days old — +30
  • Expired domain — +25
  • No domain locks — +15
  • Personal email on any contact — +10 each
  • Shared hosting nameservers — +5
  • Budget / low-tier registrar — +5
  • DNSSEC unsigned — +3

Every signal that fires is emitted in the signals array with its field name, level, and human-readable detail — making the score fully auditable.
